# Exercícios de Fixação: Explorando Outra API Pública

Agora que você já consumiu a ViaCEP e viu um exemplo com imagens, chegou a hora de testar uma API pública diferente por conta própria.

## Objetivo da atividade

Criar uma página simples que consome uma API pública e mostra o resultado na tela usando JavaScript.

## Regras da atividade

- usar `fetch()` para buscar os dados;
- mostrar o resultado diretamente na página;
- tratar erro com mensagem visível ao usuário.

## Sugestões de APIs

Você pode escolher uma destas opções:

- Dog CEO API: imagem aleatória de cachorro;
- The Cat API: imagem aleatória de gato;
- Open-Meteo: clima da cidade usando latitude e longitude;
- Rest Countries: informações de um país;
- Lorem Picsum: imagens aleatórias para uma galeria.

## Repositório para pesquisar outras opções

Se quiser buscar outra ideia, consulte:

```text
https://github.com/public-apis/public-apis
```

## Desafio principal

Escolha uma API pública e crie uma página com:

- um título explicando a proposta;
- um botão para carregar ou atualizar os dados;
- uma área para exibir o resultado retornado;
- uma mensagem de erro, caso a requisição falhe.

## Ideias de projeto

- uma galeria de imagens que muda ao clicar no botão;
- uma tela que mostra clima de uma localização;
- uma página que exibe dados de um país;
- um painel simples com informação curiosa ou aleatória.

## Checklist

- [ ] Minha página faz a requisição com `fetch()`.
- [ ] O resultado aparece na tela, não apenas no console.
- [ ] Existe tratamento de erro.
- [ ] O projeto está organizado em HTML/PHP e JavaScript.
