# README Final - Projeto da Aula

## Objetivo
Registrar a versão final do README da atividade de qualidade de processo com foco em documentação, versionamento e rastreabilidade.

## Funcionalidades planejadas
- [x] Estrutura inicial de documentação
- [x] Fluxo com branch de funcionalidade
- [x] Primeiro Pull Request da turma
- [x] Correção registrada com commit de fix

## Estrutura do projeto

```text
readme-qualidade/
  README.md
```

## Como executar o fluxo da atividade

### Pré-requisitos
- Git instalado
- Conta no GitHub

### Passos
1. Criar a pasta do projeto e iniciar o Git.
2. Criar branch de documentação.
3. Criar branch de funcionalidade.
4. Abrir Pull Request no GitHub.
5. Finalizar com branch de correção.

## Fluxo de Pull Request
1. Subir a branch de funcionalidade:

```bash
git push -u origin feature/primeira-funcionalidade
```

2. Abrir Pull Request com base em main.
3. Revisar título e descrição.
4. Se disponível, ativar Enable auto-merge.
5. Se não estiver disponível, concluir com Merge pull request.

??? tip "Conexão com qualidade"
    Cada branch e cada commit ajudam a comprovar rastreabilidade e organização do processo de desenvolvimento.

## Controle de Qualidade
### Testes planejados
- [ ] Validar estrutura do README
- [ ] Validar padrão de commits
- [ ] Validar abertura de Pull Request
- [ ] Validar merge na branch principal

### Padrões de commit
- docs: mudanças em documentação
- feat: nova funcionalidade
- fix: correção de defeito

Exemplos:
- docs: cria readme inicial com escopo da atividade
- feat: adiciona seção de fluxo de pull request
- fix: corrige descrição do objetivo

## Padrões de Markdown
- Títulos com hierarquia clara
- Listas curtas e objetivas
- Blocos de código com linguagem declarada
- Texto direto e didático
