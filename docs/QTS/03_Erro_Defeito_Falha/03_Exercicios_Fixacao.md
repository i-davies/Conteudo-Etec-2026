# Exercícios de Fixação: Erro, Defeito e Falha

## Exercícios

??? example "Desafio 1: classificando situações"
    Para cada caso abaixo, escreva se ele representa erro, defeito, falha ou uma sequência entre eles:

    - O código foi salvo sem a validação.
    - O desenvolvedor esqueceu de validar senha vazia.
    - O usuário clicou em entrar e o sistema aceitou senha vazia.

??? example "Desafio 2: script com operador incorreto"
    Crie `comparacao.py` na pasta `QTS/03_aula/erros_defeitos_falhas`.

    Requisitos:

    - declarar uma variável `nota`;
    - criar uma condição com operador inadequado;
    - mostrar uma mensagem incorreta na tela;
    - depois corrigir o script e registrar qual era o defeito.

??? example "Desafio 3: script com erro de tipo"
    Crie `cadastro.py`.

    Requisitos:

    - guardar idade como texto;
    - tentar somar esse valor com um número inteiro;
    - explicar por que a falha aconteceu;
    - ajustar o código para funcionar corretamente.

??? example "Desafio 4: relatório curto"
    Em um arquivo `analise.md` ou no caderno, responda:

    - Qual a diferença entre defeito e falha?
    - Por que testar scripts pequenos em Python ajuda no aprendizado de QTS?

## Checklist

- [ ] Criei ao menos dois scripts com problemas propositais.
- [ ] Observei o comportamento do programa em execução.
- [ ] Corrigi os scripts depois da análise.
